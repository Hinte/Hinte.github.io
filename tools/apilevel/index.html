Definitions

minSdk
The minimum API level required for your app to run. Devices with lower API levels cannot install the app.
targetSdk
The API level your app is designed and tested against. This enables compatibility behaviors on newer Android versions.
compileSdk
The API level used to compile your app. This determines which Android APIs are available during development.
Build.VERSION.SDK_INT
A runtime constant representing the API level of the device running your app. Use this to conditionally execute code based on the device's Android version.

Notes

Cumulative Usage represents the percentage of devices running that API level or higher. Data is based on devices visiting the Google Play Store.

Version Code constants are available in android.os.Build.VERSION_CODES.
